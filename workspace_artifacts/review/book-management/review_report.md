## 3. 问题与建议

### 3.1 高优先级

| 问题 | 描述 | 建议 |
|------|------|------|
| **后端代码缺失** | `code/book-management/backend/server.js` 文件不存在 | 请确认后端代码是否已提交，或检查正确的后端文件路径 |
| **API 基础路径硬编码** | API_BASE 硬编码为 '/api/books'，缺乏环境区分 | 建议使用配置文件或环境变量管理不同环境的 API 地址 |

### 3.2 中优先级

| 问题 | 描述 | 建议 |
|------|------|------|
| **价格验证缺失** | 价格字段没有最小值验证（虽然 HTML 有 min="0"） | 建议在 JavaScript 中增加服务端和客户端双重验证 |
| **输入长度限制缺失** | 书名、作者等字段没有最大长度限制 | 建议添加 maxlength 属性，防止恶意输入 |
| **错误处理不统一** | 有的错误显示在表格内（loadBooks），有的显示在消息区（saveBook） | 建议统一错误展示方式 |

### 3.3 低优先级

| 问题 | 描述 | 建议 |
|------|------|------|
| **搜索仅支持标题** | 当前搜索只能匹配书名 | 可考虑扩展搜索范围至作者、ISBN |
| **没有分页** | 数据量大时可能影响性能 | 建议添加分页功能 |
| **价格精度问题** | toFixed(2) 可能导致浮点数精度问题 | 建议使用专门的金额计算库（如 accounting.js） |
| **CSS 重复选择器** | 部分样式可合并优化 | 可使用 CSS 变量管理主题色 |
