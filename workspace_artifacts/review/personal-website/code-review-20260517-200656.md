## 详细发现

### 🔴 高风险

#### 1. CORS 配置允许所有来源
**文件**: `code/personal-website/backend/controller/BlogController.java`  
**行号**: 17

```java
@CrossOrigin(origins = "*")
```

**问题描述**:  
`@CrossOrigin(origins = "*")` 允许任意来源的跨域请求，这在生产环境中存在严重安全风险。攻击者可以通过恶意网站向 API 发起请求。

**建议**:
```java
// 生产环境应指定具体允许的域名
@CrossOrigin(origins = {"https://yourdomain.com", "https://www.yourdomain.com"})
```

或在 application.properties 中全局配置：
```properties
spring.web.cors.allowed-origins=https://yourdomain.com
```

---

#### 2. 缺乏输入验证与授权机制
**文件**: `code/personal-website/backend/controller/BlogController.java`  
**行号**: 44-48, 56-60

```java
@PostMapping
public ResponseEntity<BlogPost> createPost(@RequestBody BlogPost post) {
    // 直接反序列化，无任何验证
}

@DeleteMapping("/{id}")
public ResponseEntity<Void> deletePost(@PathVariable Long id) {
    // 无权限检查，任何人可删除
}
```

**问题描述**:  
- `createPost` 直接接受客户端提交的 `BlogPost` 对象，没有字段验证
- `deletePost` 方法没有权限检查，任何经过 CORS 的请求都可以删除文章

**建议**:
```java
@PostMapping
public ResponseEntity<BlogPost> createPost(@Valid @RequestBody BlogPost post) {
    // 使用 @Valid 注解触发 Bean Validation
}

// 添加权限检查
@DeleteMapping("/{id}")
@PreAuthorize("hasRole('ADMIN')")
public ResponseEntity<Void> deletePost(@PathVariable Long id) {
    // 权限验证后删除
}
```

在 `BlogPost` 类中添加验证注解：
```java
public class BlogPost {
    @NotBlank(message = "标题不能为空")
    @Size(max = 200)
    private String title;
    
    @NotBlank(message = "内容不能为空")
    private String content;
}
```

### 🟡 中风险

#### 3. CDN 资源缺少 SRI 完整性校验
**文件**: `code/personal-website/frontend/index.html`  
**行号**: 6-8

```html
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap" rel="stylesheet">
<link href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css" rel="stylesheet">
```

**问题描述**:  
外部 CDN 资源可能遭受供应链攻击或被篡改，缺少 Subresource Integrity (SRI) 校验。

**建议**:
```html
<!-- 添加 integrity 和 crossorigin 属性 -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap" crossorigin>
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css" integrity="sha512-DTOQO9RWCH3ppGqcWaEA1BIZOC6xxalwEsw9c2QQeAIftl+Vegovlnee1c9QX4TctnWMn13TZye+giMm8e2LwA==" crossorigin="anonymous">
```

#### 5. 可访问性问题 - 移动端菜单缺乏 ARIA 支持
**文件**: `code/personal-website/frontend/index.html`  
**行号**: 26-28

```html
<nav class="nav-menu" id="navMenu">
    <!-- 移动端 hamburger 按钮的 aria 属性不完整 -->
```

**问题描述**:  
hamburger 按钮缺少 `aria-expanded` 和 `aria-controls` 属性，菜单本身也缺少 `aria-label`。

**建议**:
```html
<nav class="nav-menu" id="navMenu" aria-label="主导航">
    ...
</nav>

<button class="hamburger" id="hamburger" aria-label="菜单" aria-expanded="false" aria-controls="navMenu">
```

---

#### 6. 字体加载可能阻塞渲染
**文件**: `code/personal-website/frontend/index.html`  
**行号**: 6-7

**问题描述**:  
Google Fonts 同步加载可能阻塞首屏渲染。

**建议**:
```html
<!-- 添加预连接 -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

<!-- 异步加载字体 -->
<link rel="preload" href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap" as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript>
    <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap">
</noscript>
```
